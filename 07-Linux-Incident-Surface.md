# 07 - Linux Incident Surface

## What I Learned
- How to investigate a compromised or suspicious Linux system
- Exploring different areas of "Incident Surface" in Linux
- Identifying attacker footprints using logs and system artifacts
- Checking running processes, network connections, persistence mechanisms
- Analyzing important log files during incident investigation

## Key Takeaways
- Understood how attackers leave traces on Linux systems
- Learned practical investigation techniques using logs and commands
- Realized the importance of quick log analysis during security incidents

## Most Useful Techniques Learned
- Checking running processes (`ps aux`)
- Checking network connections (`netstat`, `ss -tuln`)
- Analyzing authentication logs
- Looking for suspicious files and cron jobs

## Tools / Concepts Covered
- Linux Incident Investigation
- Process Monitoring
- Network Connection Analysis
- Log-based Forensics
